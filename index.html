Manual de Operações (Mission Control)

Arquitetura Baseada em Versões

O Mission Control funciona com um sistema de Entidades Hierárquicas. Isto significa que tudo o que você regista (uma partida, uma falha, uma nova peça) está "amarrado" a uma Versão Específica do robô. Assim, você consegue provar com dados exatos se a "Garra v2" melhorou a eficiência em relação à "Garra v1".

Estrutura da Árvore de Dados no Firebase

1. O Workflow Correto

Siga este ciclo de desenvolvimento para manter o seu banco de dados sempre preciso e limpo:

  • Blueprint: Comece por criar uma Nova Versão do Robô (Ex: "v1.0").
  • DevOps: Registe as modificações e as peças que testou nessa versão.
  • Partidas: Jogue! Os pontos vão diretamente para a versão ativa no topo da tela.
  • Analytics: Veja os gráficos para saber se precisa criar uma "v1.1".

2. O Seletor Global (Topo)

O menu suspenso no topo é a "Máquina do Tempo" do sistema.

  • Tudo o que for salvo (partidas ou engenharia) será guardado apenas na versão que estiver selecionada lá em cima.
  • Troque a versão no seletor e todos os gráficos e tabelas atualizarão magicamente para refletir o passado.

3. Magia do "Puxar Deployed"

Na aba Blueprint, o botão "Puxar Dados da Nuvem" constrói o seu robô automaticamente.

  • Ele varre toda a base de dados global de engenharia.
  • Ignora os testes e puxa apenas as peças marcadas com o status Deployed (Competição).
  • Assim, você não precisa digitar nada para montar o BOM da nova versão.

4. Compare (Comprovar Evolução)

Excelente para justificar escolhas de design para os juízes do prêmio Innovate/Design.

  • Selecione a versão antiga (A) e a nova (B).
  • O painel destaca em verde os atributos que melhoraram (ex: ficou mais leve) e em vermelho o que piorou (ex: ficou mais caro).
Dica de Ouro do MetalLab: Nunca misture partidas de duas configurações mecânicas diferentes na mesma versão. Se você mudou a relação de engrenagens do braço, vá no Blueprint, crie a versão "v1.2" e grave as novas partidas lá. Isso garante que as estatísticas do Analytics não fiquem contaminadas!
